## 전체 개요

### 시스템 목적
건설사가 고객용 옵션 선택 시스템을 기획하고 구성하기 위한 도구입니다.

### 3단계 프로세스
```
┌─────────────┐      ┌──────────────┐      ┌──────────────┐
│ 체크리스트   │  →  │   모듈 확인   │  →  │  프로토타입   │
│  (35개 질문) │      │ (43개 모듈)  │      │    (미리보기)  │
└─────────────┘      └──────────────┘      └──────────────┘
```

### 데이터 흐름
1. **체크리스트 작성**: 사용자가 35개 질문에 답변
2. **"모듈 확인" 버튼 클릭**: `calculateRequiredModules()` 실행
3. **모듈 자동 계산**: 답변 기반으로 필요한 모듈 자동 선택
4. **모듈 조정**: 사용자가 선택 모듈 추가/제거 가능
5. **프로토타입 확인**: 선택된 모듈 기반 프로토타입 미리보기

## 상태 관리 (Zustand Store)

### Store 구조
```typescript
{
  checklistAnswers: ChecklistAnswers,  // 35개 질문 답변
  selectedModules: string[],           // 선택된 모듈 배열 ['M01-1', 'M02-2', ...]
  activeTab: 'checklist' | 'modules' | 'prototype'
}
```

### 주요 액션

| 액션 | 설명 | 파라미터 |
|------|------|----------|
| `setAnswer` | 질문 답변 저장 | `(questionId, value)` |
| `calculateRequiredModules` | 답변 기반 모듈 자동 계산 | 없음 |
| `toggleModule` | 모듈 선택/해제 토글 | `(moduleCode)` |
| `setActiveTab` | 활성 탭 변경 | `(tab)` |
| `resetChecklist` | 체크리스트 및 모듈 초기화 | 없음 |

### 데이터 타입

```typescript
// 질문 ID 예시
type QuestionId = 'q1_1' | 'q1_2' | 'q1_3' | ... | 'q9_4'

// 답변 타입
type AnswerValue = boolean | string | string[]

// 모듈 ID 예시
type ModuleId = 'M01-1' | 'M01-2' | ... | 'M08-3'
```

## 사용자 플로우

### 1️⃣ 체크리스트 작성 단계

```
사용자 → 질문 답변 → Store.setAnswer() → checklistAnswers 업데이트
```

**주요 동작:**
- 질문 답변 시마다 실시간으로 store 업데이트
- 조건부 질문 자동 표시/숨김 (예: Q1-3은 Q1-2 = Y일 때만 표시)
- 섹션 펼치기/접기 가능

**버튼:**
- **선택 초기화**: `resetChecklist()` 호출
- **모듈 확인**: `calculateRequiredModules()` + `setActiveTab('modules')` 호출

### 2️⃣ 모듈 확인 단계

```
"모듈 확인" 클릭 → calculateRequiredModules() → selectedModules 업데이트 → 모듈 탭으로 이동
```

**주요 동작:**
- 자동 계산된 모듈이 체크된 상태로 표시
- 필수 모듈은 체크 해제 불가 (체크박스 비활성화)
- 선택 모듈은 수동으로 추가/제거 가능 (`toggleModule()` 호출)
- 실시간 통계 표시 (총 모듈 수, 필수 모듈 수, 선택 모듈 수)

**버튼:**
- **이전**: 체크리스트 탭으로 이동
- **프로토타입 보기**: 프로토타입 탭으로 이동
- **프리셋 저장**: (향후 구현)

### 3️⃣ 프로토타입 미리보기 단계

```
"프로토타입 보기" 클릭 → setActiveTab('prototype') → 선택된 모듈 기반 UI 미리보기
```

**주요 동작:**
- `selectedModules` 배열 기반으로 프로토타입 조건부 렌더링
- 선택된 모듈에 따라 UI 요소 동적 표시/숨김

### 탭 전환 흐름도

```mermaid
graph LR
    A[체크리스트] -->|모듈 확인 버튼| B[모듈 확인]
    B -->|프로토타입 보기 버튼| C[프로토타입]
    B -->|이전 버튼| A
    C -->|모듈로 이동 버튼| B
```

### 전역 상태 동기화

모든 컴포넌트가 Zustand store를 공유하므로:
- ✅ 탭 이동 시 데이터 보존
- ✅ 뒤로가기 후 수정 가능
- ✅ 실시간 동기화 (store 변경 → 모든 컴포넌트 자동 업데이트)
